In [30]:
disk_engine = create_engine('sqlite:///NYC_311.db')
start = dt.datetime.now()
chunksize = 100
j = 0
index_start = 1

for df in pd.read_csv('311_Service_Requests_from_2010_to_Present.csv', chunksize=chunksize, iterator=True, encoding='utf-8', header=0):# head=0 specifies the number of the row where the header is located    
    df = df.rename(columns={c: c.replace(' ', '') for c in df.columns}) # Remove spaces from columns

    df['CreatedDate'] = pd.to_datetime(df['CreatedDate'], errors='coerce', infer_datetime_format=True) # Convert to datetimes
    df['ClosedDate'] = pd.to_datetime(df['ClosedDate'], errors='coerce', infer_datetime_format=True)

    df.index += index_start

    # Remove the un-interesting columns
    columns = ['Agency', 'CreatedDate', 'ClosedDate', 'ComplaintType', 'Descriptor',
               'CreatedDate', 'ClosedDate', 'TimeToCompletion', 'City']

    for c in df.columns:
        if c not in columns:
            df = df.drop(c, axis=1)    
    
    j+=1
    if j%12000 == 0:
        print('{} seconds: completed {} rows'.format((dt.datetime.now() - start).seconds, j*chunksize))

    df.to_sql('data', disk_engine, if_exists='append') # Connect the trunk to the database
    index_start = df.index[-1] + 1

def mem_usage(pandas_obj):
    if isinstance(pandas_obj,pd.DataFrame):
        usage_b = pandas_obj.memory_usage(deep=True).sum()
    else: # we assume if not a df it's a series
        usage_b = pandas_obj.memory_usage(deep=True)
    usage_mb = usage_b / 1024 ** 2 # convert bytes to megabytes
    return "{:03.2f} MB".format(usage_mb)

By looking at the first few records of the dataframe, we may find that a lot of memory space is actually wasted. For example, int64 can save values from (-9,223,372,036,854,775,808 to +9,223,372,036,854,775,807) or ($-2^{64}$ to $2^{64}-1$) and uses 8 bytes. Obviously, it is too much to store the "Year" column. Instead int16 ranges from (-32,768 to +32,767) or ($-2^{16}$ to $2^{16}-1$) is quite enough for it and uses only 2 bytes. It inspires us to transform the data to a "proper" data type that uses as little memory as possible.

Fortunatelly, pd.to_numeric offers an option downcastto detect the least possible datatype to store the data automatically. Here is the description from the documentation:

downcast : {'integer', 'signed', 'unsigned', 'float'} , default None If not None, and if the data has been successfully cast to a numerical dtype (or if the data was numeric to begin with), downcast that resulting data to the smallest numerical dtype possible according to the following rules:

  • 'integer' or 'signed': smallest signed int dtype (min.: np.int8)
    • 'unsigned': smallest unsigned int dtype (min.: np.uint8)
    • 'float': smallest float dtype (min.: np.float32)

Let's investigate its effect on each datatype one by one. First,we look at the integer type.

Before we apply this method to the whole dataset, there is a sidenote you need to pay attention to. Since the categorical column stores all of the raw string values in addition to the integer category codes, it will take more storage if we transform an object series with more than 50% unique values. Thus, we need to check each column before we transform them:

converted_obj = pd.DataFrame()
for col in DT_obj.columns:
    num_unique_values = len(DT_obj[col].unique())
    num_total_values = len(DT_obj[col])
    if num_unique_values/ num_total_values < 0.5:
        converted_obj.loc[:,col] = DT_obj[col].astype('category')
    else:
        converted_obj.loc[:,col] = DT_obj[col]

Parallel Computing using joblib¶

joblib is a module develped by the same team that develops scikit learn. It is based on multiprocessing module but provides more powerful and flexible methods to make your computing parallel. Let's start with a toy example first:

In [17]:
from joblib import Parallel, delayed
import numpy as np
[np.power(i,2) for i in range(10)]
Out[17]:
[0, 1, 4, 9, 16, 25, 36, 49, 64, 81]

To parallelize it, you just need to modify the syntax by a little bit using Parallel and delayed.

In [18]:
Parallel(n_jobs=4)(delayed(np.power)(i,2) for i in range(10))
Out[18]:
[0, 1, 4, 9, 16, 25, 36, 49, 64, 81]

Here we specify 4 workers to do the job simultaneously and wrap the function we want to excecute by delayed.

What about multiple input and multiple output? Parallel will collect the returns of the function into a list.

In [19]:
num_pair = [(1,2), (2,3), (0,5)]
def foo(a,b):
    return a**2, b**2
Parallel(n_jobs=4)(delayed(foo)(i,j) for i,j in num_pair)
Out[19]:
[(1, 4), (4, 9), (0, 25)]

Next, let's try the bootstrap example we've already seen using R. First, we load the Iris data and subset only one feature (sepal length) of the data.

from sklearn.linear_model import LogisticRegression
def booti(i, X, Y):
    np.random.seed(i)
    ind = np.random.choice(range(X.shape[0]), 100, replace=True)
    clf = LogisticRegression(random_state=i, solver='lbfgs').fit(X[ind].reshape([-1,1]), Y[ind])
    return np.vstack((clf.intercept_, clf.coef_))

Compared with the original 15s, we do accelearate the procedure by about twice. But why is it not 4 times? The reason is that though we have 4 workers at the same time, we need to pay some extra time on initialization, communication, and collection of the output of each worker. These extra time that is not related to the actual function of the program is called "overhead".

You can enable verbose option to output the detailed running information. The larger value verbose is, the more frequent the middle step information will be printed:

Another useful trick is to put the data into a memmap format. A memory mapped file is just a chunk of memory in your program whose bytes are accessible by more than one process. One reason to do that is to save the "overhead" time. The other reason is to avoid replicating the data for too many times. It is extremely useful when you data is too big and the memory can't hold so many copies of the same data.

We illustrate its effect of saving "overhead" by running the bootstrap for more times.

Web Scraping using Beautiful Soup¶

This part is inspired by https://www.digitalocean.com/community/tutorials/how-to-scrape-web-pages-with-beautiful-soup-and-python-3. We are going to collect all the artist names listed on the webpage together with their links.

Before you scrape, always check if they have a public API first. For example, Yelp and Twitter have their data available in a nice format. The next thing you need to do is to check the policy on data scraping of the website. Most of the websites have the anti-robot system and usually have some limit on the use of the data on the website. You definitely don't want to be involved into a lawsuit like this.

If you decide to scrape a static website, try to save the webpages to the local disk first. It avoids requesting the pages again and again during debugging.

The next step is to locate the content you want to scrape on the html file of the webpage. You can try inspect the page in your browser. Searching carefully, we find that our target texts are wrapped in <div class="BodyText"> and always start with <a.

We can fetch this kind of data using soup.find:

artist_name_list = soup.find(class_='BodyText')
artist_name_list_items = artist_name_list.find_all('a')

The target link can be fetched using artist_name.get('href')

However, the navigation part on the bottom of the page also meets the search condition, yet we don't want to include it. So we need to exclude it before we start searching.

last_links = soup.find(class_='AlphaNav')
last_links.decompose()

Lastly, we want to write the data we scraped into .csv file and save it. We use the csv.writer from the module csv.

The full code of the above procedure is as follows:

In [ ]:
for item in filenames:
    with open(item, 'r') as f:
        c = f.read()
    soup = BeautifulSoup(c, 'html.parser')

    last_links = soup.find(class_='AlphaNav')
    last_links.decompose()

    # Create a file to write to, add headers row
    with open('z-artist-names.csv','w') as outfile:
        f = csv.writer(outfile)
        f.writerow(['Name', 'Link'])

        artist_name_list = soup.find(class_='BodyText')
        artist_name_list_items = artist_name_list.find_all('a')

        for artist_name in artist_name_list_items:
            names = artist_name.contents[0]
            links = 'https://web.archive.org' + artist_name.get('href')
            # Add each artist’s name and associated link to a row
            f.writerow([names, links])
